the structure of http{
  contains req and res  
}
requests{
    core elements can be expected are:
    1:http method:such as get ,post 
     this specify the intend of the request. are used in accurately routing and processing requests
    
    2:PATH:
    The path denotes the path of the resource relative to the root URL. For example, making a GET request to https://sana.com/api/lessons would strip common elements such as the protocol (https://) and domain (codecademy.com), leaving the path of /api/lessons.
  
   3:http protocol version:
   4:headers:Headers are optional and are used to convey additional information that may be important in processing a request by a server.

5:body:
    The body contains data required to be sent to the server to process a request.
    The body is not leveraged for all request types. It is most common to see a body attached to requests with verbs such as POST, PUT, and PATCH.
}

responses :compreised of similar to those of req with a few diff 
the core elements {
  
  [1]:http protocol version:
  [2]:status code:The status code indicates whether or not the request was successful and why.
  [3]headers:similar to requests
  [4]:body:The body of a response contains data corresponding to the fetched resource. The body is optional and contains data only when necessary to fulfill the request.
} 

the movement of http{
    HTTP requests/responses contain all of the information needed to communicate with other entities on the web efficiently, but how do these messages actually make it to their destinations?

    sotransport protocols comes into the picture:
    wehave tcp and udp protocol
    tcp{
        TCP stands for Transmission Control Protocol and allows two hosts to connect and exchange data streams, guaranteeing the delivery of data packets in the same order as they were sent.
    }
    udp{
        connectionless communication model requiring no handshaking
         other applications that want to prioritize transmission speed and efficiency over security and reliability may leverage UDP 
    }
    tls{
        it is security protocol designed to facilitate secure data transmission via encryption
        TLS evolved out of the encryption protocol known as SSL (Secure Sockets Layer),
        Using TLS with HTTP will allow you to use HTTPS (Hypertext Transfer Protocol Secure), which helps denote the presence of the extra security.
    }
    different versions of http
    {
      http/1.1
       It operates by sending messages in the form of text. HTTP/1.1 is commonly used over TCP and is the slowest of the HTTP versions regarding data transmission.

     http 2(
        major revision of http 1
        developed with the intent to try and reduce web page load latency
         This also allows a single connection to be made between two communicating entities rather than multiple as required by HTTP/1.1. Similar to HTTP/1.1, HTTP/2 also leverages TCP for transport.
     )
